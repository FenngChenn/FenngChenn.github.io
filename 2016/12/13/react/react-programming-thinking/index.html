Thinking in React

  1. 将UI拆分成组件中的一个个层级
    • 首先在设计师交出原型图时，可以和设计师确定这个页面可以分为哪几部分并命名，虽然此步骤非常简单，但是这相当于给我们建立了大纲
  2. 先建立一个react静态版本
    • 需要使用props来建立静态版本而不需要使用state，state是为了时间变化的数据所准备的。组件中的层级是父子关系，props是一种从父级传递数据到子级的方式。
    • 当项目较简单时，可以自顶向下写层级关系，当项目较为复杂时，采用自底向上的方法更为清晰明了

props vs state

  • props作为从父级传递子级数据的方式，仅仅适合于静态文件，而当页面中数据需要变动时不适用

3.确定最小的UI State表达

这三个问题则是判断是不是state的关键点
(1).它是通过props从父级传递来的吗？如果是，它可能不是 state。
(2).它随时间变化吗？如果不是,它可能不是 state。
(3).你能基于其他任何组件里的 state 或者 props 计算出它吗？如果是,它可能不是state.

4.确定state应该在哪里

确定哪个组件可以修改或者拥有着这个state
记住：React在组件层级中是单向数据流动的

  • 确定哪些组件是根据state来渲染内容的
  • 找到一个所有需要state的组件的父级，这个父级必须要在目标组件之上，并且只有一个
  • 如果找不到这样一个更高的层级，那么就简单的创建一个新的组件，并把它添加到比共同拥有者组件更高的层级上

在共同拥有者处添加getInitialState方法, 通过props传递state, 而组成初始state的有filterText和inStockOnly两个属性
