componentWillReceiveProps

当一个组件收到新的props时被调用.这个方法不会为最初的渲染调用

注意：在这个生态方法中代码执行的一个最常见的错误就是设想到props会改变，接下来的一篇文章会为你解释，A implies B does not imply B implies A

componentWillReceiveProps这篇文章陈述了componentWillReceiveProps方法将会在当props的变化作为渲染的结果时被调用，一些人就设想这就意味着如果componentWillReceiveProps被调用，那么props肯定改变了，但是这个结论逻辑上是不对的。

A implies B does not imply B implies A

React没有办法知道data没有改变，因此，React需要调用componentWillReceiveProps，因为组件需要被通知有新的props(即使新的props和旧的一样)。
你可能想React也许会有更聪明的检查新旧是否一样，但下面有一些关于这个观点的讨论:

  • 旧的mydata和新的mydata真的是同样的物理对象(只有对象的内部指向值改变了)，由于参照了triple-equals-equal，进行一个平等检查时并不会告诉我们这个值改变了，唯一可能解决的方法就是创建一个数据的深拷贝，然后做一个深比较—-但是对于大数据结构来说过分的昂贵了(特别是循环)。
  • mydata对象可能包括对在闭包中捕捉到变量的函数的引用，React没有办法观察到这些闭包，也因此没有办法复制它们或者查证它们相等不相等。
  • mydata对象可能包含在父组件渲染的时候重复实例化对象的引用(不是triple-equals-equal)，但是他们在概念上是相等的(同样的key和value)。深比较(比较昂贵)可以探测到这些，除非这个函数又出现了问题，因为没有可靠的方法去比较两个函数观察它们语义上的相等。
