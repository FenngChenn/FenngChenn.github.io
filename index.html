react-programming-thinking

发表于 2016-12-13   |   分类于 react

Thinking in React

  1. 将UI拆分成组件中的一个个层级
    • 首先在设计师交出原型图时，可以和设计师确定这个页面可以分为哪几部分并命名，虽然此步骤非常简单，但是这相当于给我们建立了大纲
  2. 先建立一个react静态版本
    • 需要使用props来建立静态版本而不需要使用state，state是为了时间变化的数据所准备的。组件中的层级是父子关系，props是一种从父级传递数据到子级的方式。
    • 当项目较简单时，可以自顶向下写层级关系，当项目较为复杂时，采用自底向上的方法更为清晰明了

props vs state

  • props作为从父级传递子级数据的方式，仅仅适合于静态文件，而当页面中数据需要变动时不适用

React学习六——ReactDOM

发表于 2016-12-01

ReactDOM

react-dom包提供了特有的DOM方法，可以被应用到你app的最外层，并且作为一个输出的出口，如果你需要的话，可以在React模板外面。你的大多数组件应该需要使用这个模块。

ReactDOM.render

  • 在提供的container里向DOM中渲染ReactElement，然后向组件当中返回一个参考，或者对于无状态组件返回null
  • 如果ReactElement预先被渲染到了container中，那么在container中将会执行一个更新，必要的时候还会反映给最新的React组件去改变DOM结构
  • 如果参数callback方法存在，方法将会在组件被渲染完成或者更新完成之后执行。

Note：

  1. ReactDOM.render()控制了你传入节点的容器内容，任何里面已存在的DOM元素在第一次被调用的时候都会被替换掉，然后调用React的DOM的diff算法进行有效率的更新。
  2. ReactDOM.render()不会修改容器的代码（只会修改容器的子结构），以后，很有可能向一个存在的DOM节点插入一个组件，而不是覆盖已存在的子结构。
  3. ReactDOM.render()目前是返回一个ReactComponent根节点实例。可是，使用这个返回值是继承性质的并且应该避免，因为未来React的版本在某种情况下可能会异步渲染组件。如果你需要一个ReactComponent根实例，那么首选的解决办法就是在根元素上链接一个回调函数。

React学习五——react顶层API和children的子方法

发表于 2016-11-08

Top-Level API For React

React

  • React.DOM
    • React.DOM提供了很简洁的封装对于DOM组件，仅仅当不使用JSX语法的时候才使用这个方法，比如React.DOM.div(null, ”Hello world!”)
  • React.PropTypes
    • React.ProTypes提供很多验证器来验证传入数据的有效性
  • React.Children
    • React.Children提供多种方法来处理this.props.children的模糊数据结构。
  • React.Children.map
    • array React.Children.map(object children, function fn [, object thisArg]) 返回值为array
    • 在每一个包含于children参数中的直属子集上执行调用fn函数，this被指向为上下文。如果children是一个关键的对象或数组，那么它将会被遍历：fn将不会被传入进去包含的对象。如果children是null或者undefined那么返回的也是null或者undefined，而不是一个数组。
  • React.Children.forEach
    • React.Children.forEach(object children, function fn [, object thisArg])
    • 和React.Children.map类似但是不返回一个数组
  • React.Children.count
    • Number React.Children.count(Object children)
    • 返回children的组件总数，和传递给map或者forEach的回调函数的调用次数相同
  • React.Children.only
    • object React.Children.only(Object children)
    • 返回children中仅有的子结构，否则抛出异常

React学习四——react顶层API

发表于 2016-11-07

Top-Level API For React

React是React类库的入口。如果你正在使用预编译包的其中一个，那么React可能是一个全局变量；如果你使用的是CommonJS模块，你可以用require()引入React。
  • React.Component
    • 当你定义使用ES6语法时，这个是React组件类的基础类。
  • React.createClass createClass(必须有render方法)
    • 创建一个组件类，给一个定义。一个组件包括一个返回一个单一子结构的render方法，这个子结构也许还有一个任意深度的子结构。使得组件不同于标准原型类的一点就是你不需要用new去定义它们，实例化。它们是很简洁的封装，可以为你在后台创建实例。
  • React.createElement
    • ReactElement(string/ReactClass type自己定义的类名也行，props 属性，children 子结构)创建并且返回一个新的给定类型的ReactElement，type参数可以是一个html标签名称字符串也可以是一个ReactClass(可以创建于React.createClass)。
  • React.cloneElement
    • 克隆并且返回一个新的、用element作为开始的ReactElement，复制完的元素会继承原有元素的属性还具有它自己新的属性，新的子结构会代替原有的子结构，就是说原来的子元素也会被覆盖。不像React.addons.cloneWithProps，key和ref也会从原来的元素中被保留复制下来，在结合这些属性的时候没有特殊的情况(不像cloneWithProps)。
  • React.createFactory
    • 返回一个能生成给定类型的ReactElement的方法，像React.createElement，参数类型可以是html标签字符串类型，例如div、span等等，也可以是一个ReactClass。
  • React.isValidElement
    • 检查这个元素是不是ReactElement

let和const命令

  • ES6新增的let命令和var命令类似，但是却比var命令更加准确，可以避免一些我们平时敲代码容易犯的错误，
    • ES6规定所声明的变量只在let命令所在代码块有效
    • let不存在变量提升，一定要先声明才能使用，在声明之前打印变量或者给变量赋值都返回错误
    • 只要块级作用域中存在let命令，let所声明的变量就绑定了这个区域，不受块级作用域之外的影响
    • let不可以在相同作用域重复声明，let和let，let和var
    • 对于const来说，只声明不赋值就报错
    • const，顾名思义是定义常量的，对于单一类型的变量，一旦赋值就不能改变，而对于复合类型的变量来说，变量名不指向数据，而指向数据所在的地址，比如对象，可以赋值对象的属性，但是直接等于某个对象是错误的
