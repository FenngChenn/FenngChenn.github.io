Vue是一个构建用户界面的渐进式框架，不像其他庞大的框架一样，vue是自底向上合理增长的开发设计。
它的核心库只专注于视图层，而且非常容易去理解，并且可以和其他库或者已存在的项目进行结合。从另一方面来说，Vue也完全有能力去驱动复杂的被使用在由流行工具和支持的库所结合的单页面应用。

Thinking in React

  1. 将UI拆分成组件中的一个个层级
    • 首先在设计师交出原型图时，可以和设计师确定这个页面可以分为哪几部分并命名，虽然此步骤非常简单，但是这相当于给我们建立了大纲
  2. 先建立一个react静态版本
    • 需要使用props来建立静态版本而不需要使用state，state是为了时间变化的数据所准备的。组件中的层级是父子关系，props是一种从父级传递数据到子级的方式。
    • 当项目较简单时，可以自顶向下写层级关系，当项目较为复杂时，采用自底向上的方法更为清晰明了

props vs state

  • props作为从父级传递子级数据的方式，仅仅适合于静态文件，而当页面中数据需要变动时不适用

3.确定最小的UI State表达

这三个问题则是判断是不是state的关键点
(1).它是通过props从父级传递来的吗？如果是，它可能不是 state。
(2).它随时间变化吗？如果不是,它可能不是 state。
(3).你能基于其他任何组件里的 state 或者 props 计算出它吗？如果是,它可能不是state.

4.确定state应该在哪里

确定哪个组件可以修改或者拥有着这个state
记住：React在组件层级中是单向数据流动的

  • 确定哪些组件是根据state来渲染内容的
  • 找到一个所有需要state的组件的父级，这个父级必须要在目标组件之上，并且只有一个
  • 如果找不到这样一个更高的层级，那么就简单的创建一个新的组件，并把它添加到比共同拥有者组件更高的层级上

在共同拥有者处添加getInitialState方法, 通过props传递state, 而组成初始state的有filterText和inStockOnly两个属性

发表于 2016-12-01

ReactDOM

react-dom包提供了特有的DOM方法，可以被应用到你app的最外层，并且作为一个输出的出口，如果你需要的话，可以在React模板外面。你的大多数组件应该需要使用这个模块。

ReactDOM.render

  • 在提供的container里向DOM中渲染ReactElement，然后向组件当中返回一个参考，或者对于无状态组件返回null
  • 如果ReactElement预先被渲染到了container中，那么在container中将会执行一个更新，必要的时候还会反映给最新的React组件去改变DOM结构
  • 如果参数callback方法存在，方法将会在组件被渲染完成或者更新完成之后执行。

Note：

  1. ReactDOM.render()控制了你传入节点的容器内容，任何里面已存在的DOM元素在第一次被调用的时候都会被替换掉，然后调用React的DOM的diff算法进行有效率的更新。
  2. ReactDOM.render()不会修改容器的代码（只会修改容器的子结构），以后，很有可能向一个存在的DOM节点插入一个组件，而不是覆盖已存在的子结构。
  3. ReactDOM.render()目前是返回一个ReactComponent根节点实例。可是，使用这个返回值是继承性质的并且应该避免，因为未来React的版本在某种情况下可能会异步渲染组件。如果你需要一个ReactComponent根实例，那么首选的解决办法就是在根元素上链接一个回调函数。
